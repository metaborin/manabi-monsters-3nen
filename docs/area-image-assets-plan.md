## デザイン意図

- 国語: 本の小道、巻物、鉛筆、ことばの葉っぱで、落ち着いた森の探索感を出す。文字は入れず、無地の本や石板にする。
- 算数: そろばん風の道具、図形ブロック、定規風モチーフで、整理された数の冒険エリアにする。数字や式は入れない。
- 理科: しぜんラボ、草花、虫めがね、フラスコ、観察ノートで、発見や実験の楽しさを出す。ラベルや文字は入れない。
- 社会: 町並み、地図、橋、山、川、地球儀風モチーフで、町や地図を旅する雰囲気にする。地図上の地名や文字は入れない。
- 英語: カラフルな街角、吹き出し風モチーフ、カード、虹、星で、ポップで親しみやすい英語エリアにする。ABCなどの文字は入れない。

## 確認結果

- 背景画像5枚はすべてPNG、1600 x 900。
- カード用イラスト5枚はすべてPNG、1024 x 1024、RGBA。
- カード用イラストの軽量版5枚はすべてPNG、512 x 512、RGBA。
- カード用イラストは背景透過済み。画像端の不透明ピクセルは0で、カード上に単色背景が出にくい状態。
- 背景画像は透過なし。アプリ画面の背景や紹介エリアにそのまま使う想定。

## 後で組み込むときの注意点

1. まずは `src/data/areas.ts` に背景画像やカード画像のパスを追加し、既存の `emoji` 表示はフォールバックとして残す。
2. `public` 配下の画像URLは、既存の `publicAssetUrl()` のように `import.meta.env.BASE_URL` を通して解決する。
3. 背景画像はUI文字の下に敷く場合、半透明オーバーレイやカード背景色を併用して読みやすさを確保する。
4. ホームカードや島画面の小さな表示には、基本的に512px版のカード画像を使う。
5. 背景画像は容量が大きめなので、実装時に表示箇所が増える場合は軽量版や遅延読み込みを検討する。

今回の作業では、アプリ本体への画像組み込みは行っていない。
